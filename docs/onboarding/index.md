# Onboarding

This section will contain everything a new team member needs to get up to
speed on the iFlot project.

## What will be documented here

- Repository overview and naming conventions
- Local development setup for each repo
- Branching and pull request conventions
- Tools and access required
- Where to find decisions and architectural context
- Who owns each area of the project
- How to contribute to this documentation

## Current status

The project is in early-stage discovery and technical foundation. Onboarding
documentation will be added as the team, tooling, and processes stabilize.

For now, the best starting points are:

- [Product vision](../vision/index.md) — what iFlot is and why it exists
- [Product overview](../overview/product-overview.md) — what the system does
- [ADR index](../adr/index.md) — key decisions made so far
- [Architecture](../architecture/index.md) — technical structure
