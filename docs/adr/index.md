# Architecture Decision Records

This section contains the architecture decision records for iFlot.

Each ADR captures a decision, the context that motivated it, and its
consequences. ADRs are not updated retroactively. If a decision changes,
a new ADR is created that supersedes the previous one.

## How to read an ADR

Start with the context to understand the problem. Read the decision to
understand what was chosen and why. Check the consequences to understand
the trade-offs accepted.
